# nfd2nfc

Python で実装した、ディレクトリ内の NFD 形式ファイル名を NFC 形式に変換するスクリプトです。

## 概要

macOS では同じ文字列でも Unicode の正規化形が異なると別のファイルとして扱われます。
このスクリプトはディレクトリ内のファイル名を NFC（統一済み正規化形）に正規化し、
NFD 由来のファイル名を整理します。

## 要件

- Python 3.5+

## インストール

```bash
chmod +x nfd2nfc.py
sudo cp nfd2nfc.py /usr/local/bin/nfd2nfc
```

## 使用方法

```bash
# 特定のディレクトリを指定
nfd2nfc -d /path/to/target

# 現在のディレクトリを対象にする
nfd2nfc

# ヘルプ表示
nfd2nfc -h
```

### オプション

| オプション | 説明 |
|---|---|
| `-d <directory>` | 変換対象のディレクトリを指定 |
| `-h`, `--help` | ヘルプを表示して終了 |

オプションを指定しない場合、カレントディレクトリが対象になります。

## 動作例

```bash
$ python3 nfd2nfc.py -d ~/Documents
  café → café
  filée → filee

Renamed 2 file(s).
```

## 仕組み

Python 標準ライブラリの `unicodedata.normalize("NFC", ...)` を使って
ファイル名を NFC に正規化します。

- 指定ディレクトリ内の全ファイルを再帰的に走査
- 各ファイル名を NFC に正規化
- NFD と NFC で名前が異なる場合のみリネーム
- 変換前後の名前を出力

## なぜ NFD が発生するのか

macOS ではファイルシステムによって Unicode の正規化挙動が異なります。

- **HFS+** — 自動的に NFD でファイル名を格納する。つまり HFS+ 上では
  `café` は `cafe` + combining accent（U+0301）の 2 バイトで保存される。
- **APFS** — NFC で格納される。HFS+ から移行したデータや
  HFS+ 上のボリュームでは NFD が残る可能性がある。
- **Finder でのコピー** — Finder がファイルをコピーする際、
  ファイルシステムに依存して NFD になる場合がある。
  HFS+ 上のボリュームから外部メディアへコピーした場合などに
  NFD 名のファイルが生成されることがある。

### 日本語の濁点・半濁点の事例

日本語の濁点（゛、U+3099）と半濁点（゜、U+309A）は NFD で
「仮名 + 濁点」という 2 文字の組み合わせになる。

| 文字 | NFC（1 文字） | NFD（2 文字） |
|---|---|---|
| ぎ | U+304E | U+304B + U+3099 |
| じ | U+3058 | U+3058 + U+3099 |
| ぶ | U+3076 | U+3073 + U+3099 |
| ぷ | U+3070 | U+3073 + U+309A |

HFS+ 上ではこれらの文字が「仮名」と「濁点」に分解されて保存されるため、
同じ文字列でもファイルシステムによって NFD/NFC が分かれる可能性がある。

このスクリプトはこうした経緯で生じた NFD 名のファイル名を
NFC に統一するために存在します。
